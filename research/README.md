# research/

External references, comparisons, and notes.

**Important**: Research material is for reference only. It is **not** the source of truth for this project. The canonical guidance lives in:

- `CONTEXT.md`
- `AGENTS.md`
- `docs/memories/`

## What belongs here
- One focused note per significant external project, standard, or discussion.
- Curated high-signal links (see LINKS.md).
- Comparison notes and "why we did X differently".
- Historical context (e.g. the original lazy-loading feature request).

## Structure
- `research/README.md` — this file
- `research/LINKS.md` — quick curated links
- `research/templates/project-note.md` — template for new notes
- `research/github-projects/` — notes about specific GitHub repos or issues
- `research/notes/` — deeper write-ups or cross-project observations

## Guidelines
- Keep entries short and specific.
- Always capture: URL, license (if relevant), stack, why it matters to us, cherry-pick / avoid.
- Do not copy code or designs blindly.
- When a research note leads to a real decision, record the decision in `docs/memories/architectural_decisions/` or `patterns_and_lessons.md` and update CONTEXT.md.
- Prefer quality over quantity. Delete stale notes.
